# FMOD Studio 導入およびナビ音声イベント作成手順書 (FMOD_INSTRUCTIONS.md)

この手順書では、Euro Truck Simulator 2 (ETS2) 用のナビ音声MODを作成するための FMOD Studio のインストールと設定方法を解説します。

---

## 1. FMOD Studio のインストール

ETS2のサウンドエンジンであるFMODに対応するため、適切なバージョンの FMOD Studio を導入する必要があります。

1. **FMODのアカウント作成:**
   - [FMOD公式ウェブサイト](https://fmod.com/) にアクセスし、右上の「Sign In」または「Sign Up」から無料のアカウントを作成します。
2. **インストーラのダウンロード:**
   - サインイン後、[FMOD Downloads ページ](https://fmod.com/download) に進みます。
   - **FMOD Studio** のダウンロードリストから、あなたのETS2のバージョンに対応するバージョンを選択します。
     - **ETS2 1.50以降〜最新 (1.56):** **FMOD Studio 2.03.xx** または **2.02.xx** 系統（SCS公式Wikiで推奨されているバージョンを選択してください）。
     - ※古いゲームバージョンを対象にする場合は、対応するFMOD Studio 2.01.xxや2.00.xxが必要です。
3. **インストール:**
   - ダウンロードしたインストーラを実行し、画面の指示に従ってインストールを完了します。

---

## 2. SCS公式サウンドテンプレートの入手

SCS Softwareが提供する公式のFMODテンプレートプロジェクトを使用します。

1. **Wikiからテンプレートをダウンロード:**
   - [SCS Modding Wiki - Sound Downloads](https://modding.scssoft.com/wiki/Documentation/Engine/Sound/Downloads) から、ゲームバージョンに対応する **Master Bank Template**（例: `fmod_template_sound_project_143_01.zip` 等）をダウンロードします。
2. **テンプレートの展開:**
   - ダウンロードした ZIP ファイルを任意の場所に展開（解凍）します。この中に `.fspro` (FMOD Studio プロジェクトファイル) が含まれています。

---

## 3. FMOD Studio でのナビ音声作成手順

### ステップ 1: プロジェクトのロード
1. FMOD Studio を起動します。
2. 展開したテンプレートのフォルダ内にあるプロジェクトファイルを開きます。

### ステップ 2: 新規バンク (Bank) の作成
> [!IMPORTANT]
> ナビ音声用のイベントは、デフォルトの `Master` バンクではなく、**専用の新規バンク**に割り当てる必要があります。
> そうしないと、ゲーム内で正しく読み込まれません。

1. 画面の **Window** メニューから **Banks** を開きます（または `Ctrl + 6`）。
2. `Master` を右クリックして、**New Bank** を選択します。
3. バンク名を入力します（例: `mym_voice`）。これがのちに `.bank` のファイル名になります。

### ステップ 3: ナビ音声イベント (Event) の作成
> [!WARNING]
> ナビ用の音声イベントは、必ず**ルートフォルダ（Eventsフォルダの直下）**に作成してください。
> サブフォルダを作成してその中にイベントを入れると、ゲームが認識できません。

1. FMODのメイン画面で **Events** タブを選択します。
2. 空いているスペースを右クリックし、**New Event** -> **Action**（または 2D Action）を選択します。
3. イベント名を、ETS2が指定する以下の規定の名前にします。
   - 例: `turn_left`, `turn_right`, `prepare_turn_left`, `start`, `finish`, `lost` など。
   - ※速度制限注意のイベント名は **`speed_signal`** が正しい名前です（`speed_limit` は認識されません）。
   - ※正確なイベント名の一覧は [requirements.md](file:///c:/Users/kohei/.gemini/antigravity/scratch/ets2_voice_mod/for_agent/requirements.md) の「4. 必要な音声イベント」を参照してください。

#### 💡 複合イベント（and_then_xxx / compound_xxx）の作成方法
ゲーム内で2段階の指示（例：「その先、左折です」）が行われるときに呼ばれる複合イベント（計14個）は、FMODのタイムライン上で既存の音声を並べるだけで綺麗に合成できます：
1. 新しくイベント（例: `and_then_turn_left`）をルートに作成します。
2. イベントのタイムライン上に、まず `and_then.wav` （その先、）をドラッグして配置します。
3. その少し後ろ（0.5秒〜1秒後など、自然に繋がる間隔）に `turn_left.wav` （ここ、左折です……っ！）を配置します。
4. これにより、再生すると「その先、左折です……っ！」と綺麗に繋がって再生されます。
5. 同様に、`compound_turn_left` イベントでは `turn_left.wav` と `and_then.wav` をタイムラインに並べます。このようにして全14個の複合イベントを作成してください。

### ステップ 4: 音声アセットの紐付け
1. 作成したイベント（例: `turn_left`）を選択し、タイムラインに音声ファイル（WAVなど）をドラッグ＆ドロップします。
   - 音声ファイルは、あらかじめ用意した `assets/` フォルダのものを利用すると管理が楽です。
2. イベントのプロパティで、ループ設定がオフになっていることを確認します（ナビ音声は1回だけ再生される必要があるため）。

### ステップ 5: ミキサールーティング (Mixer Routing) の設定
> [!IMPORTANT]
> ナビ音声のすべてのイベントは、ゲーム内のナビ用オーディオバスにルーティングする必要があります。

1. **Window** メニューから **Mixer**（または `Ctrl + 2`）を開きます。
2. 作成したすべてのナビ音声イベントを選択し、それらのルーティング（Routing）先を `/game/navigation` に設定します。

### ステップ 6: イベントをバンクに割り当てる
1. **Events** タブで作成したすべてのイベントを選択します。
2. 右クリックし、**Assign to Bank** -> 作成したバンク（例: `mym_voice`）を選択します。

### ステップ 7: バンクのビルド (Build) と GUIDs.txt の出力
1. **File** メニューから **Build**（または `F7`）を選択してビルドを実行します。
2. ビルドしただけだと GUID のファイル（`GUIDs.txt`）が出力されないことがあります。その場合は、メニューの **`File` -> `Export GUIDs...`** を選択して手動で書き出してください。
   - ※ビルド時に自動生成させたい場合は、`Edit` -> `Preferences...` -> `Build` タブの **`Write GUIDs file`** のチェックをオンにしておいてください。
3. プロジェクトのビルド出力フォルダ（通常は `Build/` フォルダ）に、以下のファイルが生成されます。
   - `<バンク名>.bank` （例: `mym_voice.bank`）
   - `GUIDs.txt`

---

## 4. MODへの組み込み

ビルドされたファイルを以下の場所にコピーします。

1. **バンクファイルの配置:**
   - 生成された `.bank` ファイル（例：`mym_voice.bank`）を `mod_root/sound/navigation/` にコピーします。
2. **GUIDs.txt のコピーとリネーム:**
   - 出力された `GUIDs.txt` をそのまま `mod_root/sound/navigation/` にコピーし、ファイル名を **`<バンク名>.bank.guids`**（例：`mym_voice.bank.guids`）にリネームします。
   - > [!CAUTION]
     > Windows環境で拡張子をリネームする際、見えないところで `.txt` が残ってしまい `mym_voice.bank.guids.txt` という名前になってしまうミスが多発します。必ず拡張子が `.bank.guids` になっているか確認してください（エクスプローラーの設定で「ファイル名拡張子」を表示する設定にすることを推奨します）。

以上の手順が完了したら、定義ファイル（`.sii`）を記述してパッケージングを行います。
